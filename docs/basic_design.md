# 基本設計書

## 1. システム概要
本システムは、複数のAIモデル（Dify経由）がDiscord上で特定の議題についてディベートを行うためのバックエンドシステムです。司会役のAIが議題を提示し、参加AIが順番に意見を述べ、最終的に司会役が結論を出します。

## 2. システム構成
- **Backend:** Laravel 13
- **AI Integration:** Dify API (各AIモデルのハブ)
- **Communication:** Discord (Interactions & Webhooks)
- **Supported Models:** Gemini, Gemma, Phi, Llama, GPT-OSS-Q2

## 3. 主要機能
### 3.1. ディベート開始機能
- Discordのスラッシュコマンド `/discuss` を受けて、新しいディベートセッションを開始します。
- 議題ごとに専用のDiscordチャンネルを自動作成します。

### 3.2. 自動ディベート進行機能
- 司会AI（Gemini）が開始メッセージを投稿します。
- 各AIが順番に発言を行います（ローテーションまたはメンションによる指定）。
- 設定された最大ターン数に達するか、司会AIが終了を判断すると議論を終了します。

### 3.3. 人間による介入機能
- スラッシュコマンド `/intervene` またはBotへのメンションにより、人間が特定のAIに指示を出すことができます。
- 介入があった場合、AIはその指示を考慮した内容で発言します。

## 4. 業務フロー
1. ユーザーがDiscordで `/discuss [topic]` を実行。
2. システムが専用チャンネルを作成し、司会AI（Gemini）に最初の発言を依頼。
3. Dify APIを通じてAIの応答を取得し、Discordへ投稿。
4. 投稿された内容から次の発言者を決定し、非同期で次のターンを処理。
5. 規定のターン数終了後、司会AIがまとめを行い終了。

## 5. 非機能要件
- **スケーラビリティ:** Laravel Queueによる非同期処理で、AIの長時間生成に対応。
- **リソース管理:** 並列実行を制限し、サーバーリソース（RAM等）を保護。
- **保守性:** ドメイン駆動設計（DDD）の考え方を取り入れたレイヤー構造を採用。
