# Release Notes

## v1.1.0 (2026-05-10)

### 新機能
- ターンIDによるジョブの排他制御を追加しました。
  - 古いターンのジョブが後から実行された場合にスキップされるようになり、議論進行の競合を防止します。
  - `debate_sessions` に `current_turn_id` を追加しました。
- Discord メッセージの分割送信に対応しました。
  - 長文の AI 応答を Discord の文字数制限に合わせて分割して投稿できるようになりました。
  - 次発言者へのメンションは最後の分割メッセージに付与されます。

### 改善
- AI 応答内の思考プロセス除去を強化しました。
  - `<think>...</think>` 形式に加え、`(think)...</think>` 形式の除去にも対応しました。
- メンション処理を改善しました。
  - Discord メンションを抽出・退避し、本文整形後に適切な位置へ付与するようにしました。
  - メンションが見つからない場合の挙動を見直しました。
- 空応答時のフォールバック処理を追加しました。
  - AI から空の応答が返った場合でも、代替メッセージを投稿して議論を継続できるようになりました。
- 次発言者が特定できない場合、司会役の Gemini にパスを回すようにしました。

### 修正
- 自己メンションやメンション漏れによって議論が停止しやすい問題を改善しました。
- 長文投稿時にメンション位置が崩れる可能性を低減しました。
- ジョブの多重実行によって古いターンが処理される可能性を低減しました。

### テスト
- ターンIDによる排他制御に関するテストを更新しました。
- Discord メッセージ分割・メンション処理に関するテストを追加・更新しました。
- 空応答および思考プロセス除去に関するテストを追加しました。

## v1.0.1 (2026-05-04)

### 修正
- `/discuss` コマンド実行時に発生していた `Non-static method App\Jobs\ProcessDebateTurn::dispatch() cannot be called statically` エラーを修正しました。
- 全てのジョブのディスパッチ処理を Laravel のグローバルヘルパー `dispatch()` を使用する形式に統一し、安定性を向上させました。

### 改善
- タイムアウトおよびリトライ間隔の設定をハードコードから `.env` で管理できるように変更しました。
  - Dify API のタイムアウト (`DIFY_API_TIMEOUT`, `DIFY_API_CONNECT_TIMEOUT`)
  - キューのリトライ間隔 (`QUEUE_RETRY_AFTER`)
  - 開始ジョブのタイムアウト (`START_DEBATE_JOB_TIMEOUT`)

## v1.0

AI Debate System の最初のメジャーリリースです。
複数の AI モデルが Discord 上で議論を行う基盤機能が完成し、人間による介入機能やドキュメントが整備されました。

## 主な新機能と改善
- **人間介入機能の強化**: `/intervene` コマンドによる指示内容が Discord 上で見えるようになり、完了済みのセッションも再開可能になりました。
- **最新環境への対応**: Laravel 13.7.0 および PHP 8.5 への対応を行いました。
- **リファクタリング**: `app` および `tests` ディレクトリ配下を整理し、型安全性の向上とコードの可読性を改善しました。
- **ログと診断の強化**: ジョブの実行状況や API リクエストのデバッグを容易にするため、ログ出力を強化し、疎通確認用コマンドを追加しました。

## ドキュメント
- **設計・運用ドキュメントの整備**: 基本設計、詳細設計、運用手順書、および Discord での使い方のガイドを追加・更新しました。
- **README の刷新**: プロジェクトの目的や貢献方法を明確化しました。
